Plugin API Reference

ImperialsBot plugins use the api object passed to the init(bot, api) method. This object provides high-level control over the bot's features and movement.

Movement
api.move(x, y, z)

Tells the bot to navigate to the specified coordinates using the pathfinder.

Movement
api.stop()

Stops all current movement and navigation. Clears control states.

Movement
api.jump(active = true)

Sets the jump control state.

Movement
api.getPos()

Returns the bot's current Vec3 position.

Combat
api.attack(entity)

Attacks the specified entity using the PvP module.

Interactions
api.breakBlock(block, direction?)

Tells the bot to dig/break the specified block.

Interactions
api.equip(item, destination?)

Equips an item from inventory to a slot (e.g., 'hand', 'off-hand').

Features
api.setKillaura(enabled)

Enables or disables the Killaura feature.

Features
api.setSpammer(enabled, config?)

Controls the Spammer feature. Config is optional.

System
api.log(message, type?)

Logs a message to the bot's console and dashboard with the plugin prefix.

System
api.chat(message)

Makes the bot send a message to the server chat.

System
api.reconnect()

Forces the bot to disconnect and reconnect.
